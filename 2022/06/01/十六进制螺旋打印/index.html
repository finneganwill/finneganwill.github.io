一道笔试题：十六进制螺旋打印

Author: Finnegan Date: 2022 / 06 / 1  15:59 Category: 一道笔试题

1. 题目

前几天看到一个题目，尝试求解了一下，写这篇博客用来记录。

2. 分析

从题目给的测试用例可以总结出一些规律：

  1. 输出的矩阵的大小是由输入的整数 n 决定的，矩阵是一个 n*n 的方针
  2. 输出方式是按 “左 - 下 - 右 - 上” 的顺序进行输出
  3. 输出的内容是一些字符串，根据字符串的顺序可以看出是顺序的 16 进制

这里总结一下求解时需要解决的几个问题：

  1. 读取输入的整数
  2. 字符串的存储
  3. 16 进制的字符型输出
  4. 输出方向/索引的变化规律

3.2 字符串的存储

由于输出的是一个矩阵，使用一个二维的字符串数组可以很好地对帖矩阵。

由于它是按16进制的大小顺序输出的，那么在处理时就可以将 16 进制 按大小保存的二维数组中，最后将二维数组原样输出即可。

这样就把螺旋打印的问题转换为了如何将字符串螺旋保存到二维数组中了，定义两个索引指针 rindex 和 cindex 分别表示行索引和列索引。

3.3 16进制的字符型输出

Java中 Integer 类有一个方法 toHexString(int i) ，这个方法可以将一个 int 类型的整数转换为 16进制的字符串表示。

由于题目中是十六进制的顺序，那么就可以使用一个计数器来不断的 +1 ，然后转换为 16 进制对应的字符。

3.4 输出方向/索引的变化规律

这个题目的难点就在于如何变换输出的索引位置。这里阐述两种方式：非递归和递归

  1. 非递归

    非递归就是按照十六进制的顺序逐个保存到二维数组中，按照 “左 - 下 - 右 - 上” 的方向。

    设置一个 int flag 来表示指向保存位置的指针前进的方向，取值分别为 1, 2, 3, 4 。

    设置两个变量 rbound 和 cbound 来表示行和列边界，当行列索引达到边界时就要更换方向，初始时两个变量的值都为 n 。

    边界条件：当指针达到边界时需要改变方向， 即修改 flag 的值和下一次运动的边界 rbound / cbound。

    指针不断变化，直至 n * n 个字符串全都保存完毕，即 count > n * n 。

  2. 递归

    一般来讲，能用递归来解决的问题必须满足两个条件：

    • 可以通过递归调用来缩小问题规模，且新问题与原问题有着相同的形式
    • 不能无限制地调用本身，须有个出口，化简为非递归状况处理。

    缩小规模： 根据螺旋输出的特点，一圈即可保存一次，通过逐圈处理可以缩小问题规模。因此缩小圈层后新问题和原问题是一样的，只是矩阵维度减小了。

    递归出口： 缩圈到最后会有两种情况：1. 只剩一个元素；2. 剩余0个元素。这可以作为递归出口。
